# 纯 LLM Latent 推理论文调研总整理

## 1. 范围与目标

这份文档把之前分散的三类材料合并到一起：

- `latent reasoning` 主线文献
- `可解释性 / SAE` 相关文献
- `steering / activation engineering` 相关文献

目标不是机械罗列所有文章，而是回答三个更实用的问题：

1. 纯 LLM latent 推理这条线已经做到哪里了？
2. 可解释性、SAE、steering 对 latent 方法到底提供了什么可靠启发？
3. 对我们后续做小中模型、可实现的 latent 方法，哪些结论最值得吸收？

---

## 2. Latent 推理主线：研究脉络

## 2.1 起点：把显式 CoT 压缩到隐空间

这条线最早的一批核心工作，重点不是“机制解释”，而是“能不能把显式推理压缩成更短、更便宜的内部连续表示”：

- [Expediting and Elevating Large Language Model Reasoning via Hidden Chain-of-Thought Decoding](https://arxiv.org/abs/2409.08561)
- [Training Large Language Models to Reason in a Continuous Latent Space](https://arxiv.org/abs/2412.06769)
- [Compressed Chain of Thought: Efficient Reasoning Through Dense Representations](https://arxiv.org/abs/2412.13171)
- [CODI: Compressing Chain-of-Thought into Continuous Space via Self-Distillation](https://arxiv.org/abs/2502.21074)

这批工作主要解决：

- 显式 CoT 太长，推理成本高
- 生成中间文本会拖慢推理
- 希望保留“多步思考”的好处，但不要把每一步都写出来

这批工作的共同特点是：

- 更关注“压缩后还能不能提分 / 省算力”
- 更少关注“压缩进去的到底是不是 faithful reasoning”

---

## 2.2 第二阶段：latent test-time scaling 与 recurrent depth

后来大量工作开始把 latent 推理看成一种 `test-time compute` 扩展方式，不再只谈压缩，而是谈“内部多想几步”：

- [Scaling up Test-Time Compute with Latent Reasoning: A Recurrent Depth Approach](https://arxiv.org/abs/2502.05171)
- [Reasoning with Latent Thoughts: On the Power of Looped Transformers](https://arxiv.org/abs/2502.17416)
- [SoftCoT: Soft Chain-of-Thought for Efficient Reasoning with LLMs](https://arxiv.org/abs/2502.12134)
- [SoftCoT++: Test-Time Scaling with Soft Chain-of-Thought Reasoning](https://arxiv.org/abs/2505.11484)
- [Continuous Chain of Thought Enables Parallel Exploration and Reasoning](https://arxiv.org/abs/2505.23648)
- [Parallel Continuous Chain-of-Thought with Jacobi Iteration](https://arxiv.org/pdf/2506.18582)

这阶段的核心变化是：

- latent 不只是“压缩显式 CoT”
- latent 也可以是“在隐空间里多迭代几轮”
- 关注点从“省 token”变成“多给内部计算预算会不会更强”

但问题也更明显了：

- 内部 step 多了，不代表真在做更好的推理
- latent depth 增加后，收益并不总是稳定

---

## 2.3 第三阶段：机制诊断与 faithful reasoning 质疑

到 2025 年末到 2026 年初，领域里开始集中质疑一个问题：

`latent token / latent step 到底是不是“真的在一步步思考”？`

代表论文包括：

- [Beyond Chains of Thought: Benchmarking Latent-Space Reasoning Abilities in Large Language Models](https://arxiv.org/abs/2504.10615)
- [Do Latent Tokens Think? A Causal and Adversarial Analysis of Chain-of-Continuous-Thought](https://arxiv.org/abs/2512.21711)
- [Do Latent-CoT Models Think Step-by-Step? A Mechanistic Study on Sequential Reasoning Tasks](https://arxiv.org/abs/2602.00449)
- [Dynamics Within Latent Chain-of-Thought: An Empirical Study of Causal Structure](https://arxiv.org/abs/2602.08783)
- [When Shallow Wins: Silent Failures and the Depth-Accuracy Paradox in Latent Reasoning](https://arxiv.org/abs/2603.03475)
- [SPOT: Span-level Pause-of-Thought for Efficient and Interpretable Latent Reasoning in Large Language Models](https://arxiv.org/abs/2603.06222)

这批论文带来的几个重要结论是：

- latent 方法可以提分，但“提分”不等于“faithful process”
- 某些 latent step 可能只是答案相关的压缩，而不是真正必要的中间计算
- 更深的 latent 迭代不一定更强，甚至会出现 `depth-accuracy paradox`
- 以后做 latent 论文，不能只汇报 accuracy，最好还要说明内部过程到底有没有更稳定、更有结构

这也是为什么我们后面更看重：

- 一致性
- 特征选择
- 正确/错误轨迹分离
- 因果必要性

## 3. 可解释性 / SAE / Steering 给我们的启发

## 3.1 什么是 activation、feature、SAE feature

- `activation`：模型某层某位置的中间数值状态，也就是 hidden state / residual stream 里的值。
- `feature`：不是单个神经元，而是 activation 空间里一个更有功能意义的方向或模式。
- `SAE feature`：用 Sparse Autoencoder 把 activation 拆开后得到的稀疏成分，便于分析、筛选、去噪。

相关论文：

- [Feature Extraction and Steering for Enhanced Chain-of-Thought Reasoning in Language Models](https://arxiv.org/abs/2505.15634)
- [CorrSteer: Generation-Time LLM Steering via Correlated Sparse Autoencoder Features](https://arxiv.org/abs/2508.12535)
- [Beyond Prompt Engineering: Robust Behavior Control in LLMs via Steering Target Atoms](https://aclanthology.org/2025.acl-long.1139/)

---

## 3.2 可靠启发 1：latent space 里确实有可利用结构

这一点已经不太需要再证明了。不同目标的论文都说明：

- 中间层表示不是完全黑箱
- 某些行为相关信号可以被提取
- 某些信号甚至能被干预，从而稳定改变输出行为

代表论文：

- [Feature Extraction and Steering for Enhanced Chain-of-Thought Reasoning in Language Models](https://arxiv.org/abs/2505.15634)
- [Steering LLM Reasoning Through Bias-Only Adaptation](https://arxiv.org/abs/2505.18706)
- [Beyond Prompt Engineering: Robust Behavior Control in LLMs via Steering Target Atoms](https://aclanthology.org/2025.acl-long.1139/)
- [Steering Knowledge Selection Behaviours in LLMs via SAE-Based Representation Engineering](https://aclanthology.org/2025.naacl-long.264/)

对 latent 方法的意义是：

`以后不必默认“只能整段 hidden state 黑箱蒸馏”，而是可以主动做 feature discovery、feature filtering、feature regularization。`

---

## 3.3 可靠启发 2：不是所有被激活的 feature 都值得信

这是最关键的一点。

很多论文都在强调：

- 某个 feature 会被输入激活
- 不等于它真的影响输出
- 更不等于它代表“真正有用的 reasoning feature”

代表论文：

- [SAEs Are Good for Steering -- If You Select the Right Features](https://aclanthology.org/2025.emnlp-main.519/)
- [CorrSteer: Generation-Time LLM Steering via Correlated Sparse Autoencoder Features](https://arxiv.org/abs/2508.12535)
- [Enhancing LLM Steering through Sparse Autoencoder-Based Vector Refinement](https://arxiv.org/abs/2509.23799)

对 latent 方法的直接启发是：

- 不要把整段 hidden state 直接当监督目标
- 不要把“和 CoT 共现的 feature”直接当作 reasoning feature
- 更稳的路线是：`先找候选特征 -> 再筛掉噪声和伪相关 -> 最后再加约束`

---

## 3.4 可靠启发 3：动态控制往往比固定全局向量更合理

后期 steering 论文已经很清楚地表明：

- 固定 steering 向量不是最优
- 固定 steering 强度也不是最优
- 不同输入、不同层，往往需要不同干预强度

代表论文：

- [Prototype-Based Dynamic Steering for Large Language Models](https://arxiv.org/abs/2510.05498)
- [In-Distribution Steering: Balancing Control and Coherence in Language Model Generation](https://arxiv.org/abs/2510.13285)
- [Activation Steering with a Feedback Controller](https://arxiv.org/abs/2510.04309)

这对 latent 方法的意义是：

`以后不要默认一个全局 latent vector 就能解决所有输入。`

---

## 4. 对我们做 latent 方法最重要的结论

综合 latent 主线和 explainability / steering 这两边，我认为最值得保留的结论有 6 条。

## 4.1 不要只看最终分数

因为 latent 论文已经反复显示：

- 能提分，不代表内部过程更 faithful
- latent step 多，不代表真的在逐步推理

参考：

- [Do Latent Tokens Think?](https://arxiv.org/abs/2512.21711)
- [Do Latent-CoT Models Think Step-by-Step?](https://arxiv.org/abs/2602.00449)
- [Dynamics Within Latent Chain-of-Thought](https://arxiv.org/abs/2602.08783)

## 4.2 不要直接监督整段 hidden state

因为整段 hidden state 会把：

- reasoning signal
- 答案线索
- 文本模板
- 长度和位置噪声

全部混在一起。

更合理的路线是：

`先选 feature，再 regularize feature。`

参考：

- [CorrSteer](https://arxiv.org/abs/2508.12535)
- [SAE-RSV](https://arxiv.org/abs/2509.23799)
- [SAEs Are Good for Steering -- If You Select the Right Features](https://aclanthology.org/2025.emnlp-main.519/)

## 4.3 一致性比“完全重合”更重要

如果同一道题有多个等价 rationale，那么更自然的目标不是让内部表示逐维完全一样，而是：

- 某些核心特征更稳定
- 某些结构关系更稳定

这就是我们后面喜欢 `Rationale-Orbit Consistency` 的原因。

## 4.4 正确/错误 latent 轨迹的差异值得利用

与其直接追求“更强 latent 推理”，不如先问：

- 正确轨迹和错误轨迹在 latent space 里能不能分开？
- 能分开的那些 feature 能不能成为后续 regularization 的对象？

参考：

- [Steer LLM Latents for Hallucination Detection](https://arxiv.org/abs/2503.01917)
- [Do Latent Tokens Think?](https://arxiv.org/abs/2512.21711)

## 4.5 小数据方向发现是现实可行的

对算力有限的情况，不应该一上来做大规模 latent pretraining。

更现实的顺序是：

1. 用少量高质量 CoT / 对错样本找 direction 或 feature
2. 先做小规模原型验证
3. 再决定是否做完整方法

参考：

- [One-shot Optimized Steering Vectors Mediate Safety-relevant Behaviors in LLMs](https://arxiv.org/abs/2502.18862)
- [Steer LLM Latents for Hallucination Detection](https://arxiv.org/abs/2503.01917)
- [Feature Extraction and Steering for Enhanced Chain-of-Thought Reasoning in Language Models](https://arxiv.org/abs/2505.15634)

## 4.6 对“causal”表述要非常保守

可以做扰动、做必要性测试、做对比分析，但不要轻易宣称：

- 这个 latent feature 就是 reasoning 本质
- 这个 latent step 已经被证明是真正因果机制

更稳妥的说法是：

- 它与正确性更相关
- 它对输出更敏感
- 它更值得被 regularize

## 6. 最后结论

如果只保留一句话，那就是：

`latent 推理这条线已经从“能不能做”进入“内部过程到底靠不靠谱、该约束什么结构、该如何避免伪推理”的阶段。`

而 explainability / SAE / steering 对我们的真正价值，不是让我们去照搬 activation hack，而是告诉我们：

- latent 空间里确实有结构
- 但不是所有结构都值得信
- 所以后续方法应该优先做特征筛选、去噪、稳定性约束，而不是粗暴对齐整个 hidden state

这正是后面 idea 候选文档的出发点。
